# caltech-course-tooltip-data

Hosted data for the [Caltech Course Tooltip](https://github.com/Varun-Rodrigues/Caltech-Course-Tooltip) Chrome extension, served via GitHub Pages so the extension can pick up data updates without a new Chrome Web Store release.

Served at: `https://varun-rodrigues.github.io/caltech-course-tooltip-data/`

## Files

- **`catalog.json`** — full course catalog (schedule, instructors, descriptions, units). Generated by the main repo's `tools/merge_data.py` pipeline.
- **`course-codes.json`** — flat array of every valid course lookup key. Generated by the same pipeline.
- **`popup-content.json`** — the extension popup's User Guide and Report a Problem text (hand-authored, edited directly in the main repo).

These are plain copies, not symlinks — GitHub does not resolve symlinks when serving file content (Pages/raw URLs serve a symlink's literal target-path text, not the referenced file), so a real copy is required for hosting to actually work.

## Publishing an update

From the main repo (`Caltech-Course-Tooltip`), regenerate the data:

```bash
pip install -r requirements.txt
python tools/catalog_processor.py
python tools/schedule_parser.py
python tools/ic_parser.py
python tools/merge_data.py
```

Then, from this repo:

```bash
./sync.sh
git add -A
git commit -m "Update catalog data"
git push
```

GitHub Pages redeploys within a minute or two of the push — no separate cache-purge step needed. The extension's background service worker picks up the new data on its next fetch (each service-worker wake attempts a fresh fetch; see `extension/background/remoteData.js` in the main repo for the fallback chain if a fetch fails).
